Aplikacja Gamifikacji dla Pracowników

Brief techniczny • 20 pracowników • Rozliczenia tygodniowe

1. Przegląd projektu

Aplikacja webowa do śledzenia i nagradzania pracowników za punktualność, obecność i osiągnięcia. System resetuje punkty co tydzień, wyświetla ranking i zarządza nagrodami.

2. Wymagania funkcjonalne

Panel Administratora

  • Zarządzanie pracownikami (dodawanie, edycja, usuwanie)
  • Definiowanie kategorii punktów (np. Obecność, Punktualność, Cele)
  • Ręczne dodawanie punktów z powodu (kategoria + opis)
  • Resetowanie punktów co piątek/poniedziałek (button + potwierdzenie)
  • Widok bieżącego rankingu tygodnia
  • Historia zmian (log kto dostał ile pkt za co i kiedy)
  • Export danych do CSV
  • Ustawienia systemu (zablokuj edycję po resecie)

Panel Pracownika

  • Logowanie (email + hasło, lub SSO jeśli dostępne)
  • Wyświetlenie swoich punktów (bieżący tydzień)
  • Tablica liderów (top 5-10 osób)
  • Historia swoich punktów (ostatnich 8 tygodni)
  • Powiadomienie email przy przyznaniu punktów

Dashboard publiczny

  • Podsumowanie tygodnia (kto wygrał, ile pkt przyznano razem)
  • Top 3 osoby z ikonkami/medalami

3. Schemat bazy danych

USERS
├── id (UUID)
├── email (string, unique)
├── password_hash (string)
├── full_name (string)
├── role (enum: admin | user)
├── created_at (timestamp)

POINTS_CATEGORIES
├── id (UUID)
├── name (string) — "Obecność", "Punktualność"
├── description (string)
├── created_at (timestamp)

POINTS_LOG
├── id (UUID)
├── user_id (FK → users)
├── category_id (FK → points_categories)
├── amount (integer)
├── reason (text)
├── given_by (FK → users) — admin who gave it
├── week_number (integer) — 1-52
├── year (integer)
├── given_at (timestamp)

WEEKLY_RESET_LOG
├── id (UUID)
├── reset_date (date)
├── points_before (JSON) — backup
├── reset_by (FK → users)

6. Szacunek pracy

  • Setup i auth: 8h
  • API endpoints: 12h
  • Frontend admin panel: 12h
  • Frontend user dashboard: 8h
  • Ranking i leaderboard: 6h
  • Email notifications: 4h
  • Testowanie i bugfixy: 10h
  • Razem: ~60 godzin (2-3 tygodnie dla 1 juniora)

7. Ważne notatki

  • Autentykacja: najpierw JWT tokens, potem opcjonalnie SSO (Google, Microsoft)
  • Email: SendGrid / Mailgun / nodemailer do powiadomień
  • Security: rate limiting, CORS, input validation
  • UI: responsive design (mobile + desktop)
  • Backup: regularny export danych
  • Monitoring: proste logging do testowania błędów

💡 Tip: Zaproponuj deweloperowi najpierw zrobić MVP bez emaili i eksplortu. Dodaj to später jako fase 2.
